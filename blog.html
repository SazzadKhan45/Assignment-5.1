What is the Document Object Model (DOM) ?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a web page as a tree of objects, where each object corresponds to a part of the page (such as an element, attribute, or text content). The DOM allows developers to manipulate the content, structure, and style of a webpage dynamically using programming languages like JavaScript.

Key points about the DOM:

1. Tree Structure: The DOM represents the document (HTML or XML) as a hierarchical tree. Each node in the tree corresponds to an element (like a div or p tag), attribute (like class or id), or piece of text within the document.

2. Dynamic Interaction: JavaScript can be used to access, modify, add, or delete elements and attributes in the DOM. This allows for dynamic content updates on a webpage without requiring a full page reload.

3. Platform and Language-Independent: The DOM is not tied to any specific programming language. However, JavaScript is most commonly used to interact with the DOM in web browsers.

4. Real-Time Changes: When a script modifies the DOM, the changes are immediately reflected in the rendered webpage, and events like clicks or mouse movements can trigger updates to the DOM.

1. getElementById()

2. getElementsByClassName()

3. getElementsByTagName()

4. querySelector()

5. querySelectorAll()

What is event delegation in the context of the DOM, and why is it useful ?

Event delegation is a programming technique used in the context of the Document Object Model (DOM) that allows you to manage events on multiple elements efficiently by attaching a single event listener to a parent element, rather than adding event listeners to individual child elements.

How it works:

When an event occurs on a child element, the event bubbles up the DOM tree (from the target element up to the root), and the event listener on the parent element can handle the event. The event listener on the parent checks which child element the event originated from, typically using the event.target property, and processes it accordingly.
